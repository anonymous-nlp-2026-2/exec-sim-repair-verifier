\section{Conclusion}
\label{sec:conclusion}

We presented the first systematic comparison of three execution-free verification formulations (\cls{}, \cwm{}, \traj{}). At function level, \cls{} and \cwm{} are statistically equivalent (TOST $p{=}0.007$, 5 seeds); \traj{} equivalence is not confirmed after multiple comparison correction. Per-test classification achieves 217$\times$ higher throughput while providing diagnostic signal. Complementary errors (oracle: 90.63\%/95.83\%; single-seed) motivate complexity-aware routing. At repository level under ${\leq}$8B LoRA, divergence reaches 18--26\pp{} ($z{=}-23.7$, 3 seeds per model), with \cwm{} falling below majority baseline---consistent across families but dependent on multiple confounds (\S\ref{sec:threats}). Three failure modes (format collapse, decision collapse, aggregation penalty) are consistent across families. \cls{} is recommended primarily for its throughput advantage and diagnostic capability; the accuracy advantage over \traj{} is directional but underpowered ($n{=}48$, MDE${\approx}$25\pp{}; Appendix~\ref{app:power}). All findings are conditioned on ${\leq}$8B LoRA fine-tuning; at larger scale, the generation bottleneck may diminish (\S\ref{sec:threats}).

Three directions follow: (1)~oracle gains motivate complexity-aware routing using the failure mode taxonomy as routing features; (2)~the semantic generation bottleneck (four interventions fail; Appendix~\ref{app:extended_analysis}(g)) suggests capacity scaling or execution trace pretraining~\cite{meta2025cwm} is needed---testing at 32B+ would resolve the primary open question; (3)~larger-scale evaluation of per-test diagnostics for downstream repair ($n{=}44$ in our pilot) would test whether \cls{}'s throughput advantage extends to repair pipelines.

\section*{Limitations}

Repo-level \cwm{} uses 3 seeds per model; the gap (18--26\pp{}) exceeds any single confound (\S\ref{sec:threats}), but correlated confounds require further replication. Our \cwm{} baseline uses ${\leq}$8B LoRA fine-tuning, not the original 32B CWM~\cite{meta2025cwm}; the generation bottleneck may diminish at larger scale. Training data comprises synthetic mutations~\cite{andrews2005mutation,just2014mutants}; synthetic-to-real transfer is untested, though the coupling effect supports relative ranking validity (Appendix~\ref{app:limitations}). The \traj{} downsample ablation uses a single model, seed, and complexity level. Model coverage is limited to 6--8B code-specialized LLMs. All evaluation uses Python-only benchmarks; languages with different test output conventions may exhibit different format compliance rates. Our evaluation targets repair verification; generalization to open-ended code generation is untested. All code, data, and model checkpoints will be released upon acceptance.
