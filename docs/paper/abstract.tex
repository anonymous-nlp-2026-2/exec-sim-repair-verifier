\begin{abstract}
Large Language Model (LLM)-based program repair generates candidate patches and selects the best via a verifier. Three execution-free alternatives to costly execution-based verification have emerged, namely per-test binary classification (\cls{}), execution output generation (\cwm{}), and trajectory-level scoring (\traj{}), yet no prior work compares them under controlled conditions. We present the first systematic comparison across two code LLM families, two complexity levels, and multiple seeds under ${\leq}$8B Low-Rank Adaptation (LoRA) fine-tuning. At function level, \cls{} and \cwm{} are statistically equivalent (Two One-Sided Tests [TOST] $p{=}0.007$, 5 seeds): classification offers no inherent advantage over generation when outputs are short. At repository level, however, this equivalence breaks down: \cwm{} accuracy drops 18--26 percentage points below \cls{} as format collapse emerges with longer, structured outputs. Preliminary 14B results narrow all gaps to ${\sim}$1 percentage point, suggesting the divergence is capacity-contingent rather than formulation-inherent. Error complementarity across formulations motivates complexity-aware routing as a key open direction.
\end{abstract}
