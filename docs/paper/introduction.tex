\section{Introduction}
\label{sec:introduction}

LLM-based automated program repair systems increasingly rely on test-time compute scaling: generating $K$ candidate patches and selecting the best via a verifier~\cite{snell2024scaling}. Full execution verification is reliable but expensive (\$1--3 per candidate at repository scale\footnote{Based on per-instance costs reported by \citet{yang2024sweagent} and \citet{xia2024agentless}.}), driving demand for execution-free alternatives. Three distinct formulations have emerged for this task: \textbf{generation-based simulation}~\cite{meta2025cwm,maimon2026selfexecution} predicts execution output strings per test; \textbf{trajectory-level scoring}~\cite{shum2025swerm} predicts a binary patch verdict without per-test decomposition; and \textbf{per-test classification}~\cite{ni2023lever,chen2023codet} predicts a pass/fail token per test. Each makes different tradeoffs between diagnostic granularity, output complexity, and inference cost.

Despite active development along each axis, no prior work provides a systematic comparison holding model capacity, training data, and compute constant. Practitioners choosing a formulation for their verification pipeline must rely on results from different models, different datasets, and different evaluation protocols. Whether one formulation is uniformly preferable, or whether the choice depends on properties of the code being verified, remains an open empirical question. A related question is whether formulation choice matters at all relative to more basic decisions: at what complexity level does fine-tuning become necessary, and does formulation selection add value beyond what fine-tuning alone provides?

These three formulations represent discrete points in the verification design space, each imposing different output complexity, diagnostic granularity, and inference cost. We present the first systematic empirical comparison across two representative code LLM families (Qwen3, DeepSeek-Coder), two complexity levels (function-level and repository-level), and multiple seeds. Our contribution is primarily methodological: a matched experimental framework that isolates formulation effects from confounding variables. The analysis also produces actionable outputs: a failure mode taxonomy that characterizes when and why each formulation fails (\S\ref{sec:repo}), error complementarity findings with practical ensemble implications (\S\ref{sec:analysis}), and formulation selection recommendations conditioned on code complexity (\S\ref{sec:conclusion}). The results reveal a three-layer structure. Fine-tuning is necessary (0-shot$\to$fine-tuned gains of 35--36\pp{} at repo level). After fine-tuning, \emph{formulation choice is governed by code complexity}: at function level, all formulations achieve comparable accuracy---consistent with the theoretical analysis in \S\ref{sec:failure-modes}, which predicts that short outputs minimize generation-specific failures. At repository level under our ${\leq}$8B LoRA protocol, this equivalence collapses; variant-level analysis reveals three distinct failure modes consistent across model families, though gap magnitude depends on multiple confounds (\S\ref{sec:threats}). Figure~\ref{fig:reformulation} illustrates the three formulations.

We make the following contributions:

\begin{itemize}[leftmargin=*,itemsep=2pt]
\item \textbf{Formulation near-equivalence at low complexity.} At function level, \cls{} and \cwm{} achieve statistical equivalence across two representative code LLM families; \traj{} reaches parity on Qwen3 but exceeds the equivalence margin on DeepSeek-Coder. Per-test classification matches trajectory-level accuracy while identifying which tests fail (\S\ref{sec:func_results}, \S\ref{sec:equivalence}).

\item \textbf{Complexity-dependent divergence.} A robust interaction ($z{=}-23.7$, $n{=}56{,}480$) confirms that formulations equivalent at function level (${\leq}$1.3\pp{}) diverge by 19--26\pp{} at repository level under ${\leq}$8B LoRA fine-tuning, driven by format collapse; variant-level analysis reveals three failure modes consistent across model families, though gap magnitude depends on single-seed evaluation and multiple confounds (\S\ref{sec:repo}, \S\ref{sec:threats}).

\item \textbf{Error complementarity.} \cls{} and \cwm{} exhibit partially complementary error patterns at function level (oracle: 90.63\%; single-seed); at repo level, oracle routing reaches 95.83\% variant accuracy (+14.58\pp{}; single-seed), the strongest practical argument for exploring the formulation design space (\S\ref{sec:analysis}).
\end{itemize}
